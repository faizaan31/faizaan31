## 🔭 Currently Exploring
I'm a **Computer Science** grad from **New York University (NYU)**, building at the intersection of:

• **Production software engineering** — backend services, distributed systems, and APIs that are reliable, observable, & built for scale.

• **Applied ML & AI** — end-to-end pipelines from data to training to real-time inference & building with modern AI in mind.

• **Quantitative rigor** — Drawn to problems where we model uncertainty, work with noisy time‑dependent data & care about both statistical rigor and system performance.

Recently I've been particularly interested in large-scale backend services, ML systems that turn raw data into real-time decisions, retrieval and intelligence systems, and performance-aware engineering where latency and robustness matter.

## 🧠 How I Think Around
I tend to approach systems from a **measurement-first perspective**. Before writing code, I usually think about:

• Latency distributions and throughput limits  • Model calibration and prediction behavior  • System observability and failure modes, and how data distributions shift over time  

This mindset comes from working across **production systems, data pipelines, and machine learning workflows**.
